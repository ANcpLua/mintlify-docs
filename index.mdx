# ANcpLua Framework

A comprehensive .NET development ecosystem providing opinionated defaults, automated polyfills, custom analyzers, and Roslyn utilities for building high-quality applications.

## What is ANcpLua.NET.Sdk?

ANcpLua.NET.Sdk is an opinionated MSBuild SDK that automatically configures your .NET projects with modern C# features, code quality analyzers, and best practices. Add it to your project and get immediate access to enforced coding standards, automated polyfills for legacy frameworks, and production-ready tooling.

## Which SDK Should I Use?

Choose the SDK variant based on your project type:

| SDK Variant | Use For | Key Features |
|-------------|---------|--------------|
| `ANcpLua.NET.Sdk` | Libraries, Console Apps, Workers | Analyzers, Polyfills, Banned APIs, Guard Clauses |
| `ANcpLua.NET.Sdk.Web` | ASP.NET Core Web APIs | All Base features + OpenTelemetry, Health Checks, HTTP Resilience |
| `ANcpLua.NET.Sdk.Test` | xUnit Test Projects | All Base features + xUnit v3 with MTP, Test Coverage |

## What Does the SDK Provide?

### Base SDK Features (ANcpLua.NET.Sdk)

| Feature | Description |
|---------|-------------|
| **Polyfills** | Automatic Index/Range, IsExternalInit for netstandard2.0/net472 |
| **Analyzers** | Pre-configured [ANcpLua.Analyzers](/analyzers/overview), Meziantou.Analyzer |
| **Banned APIs** | Enforced restrictions on [legacy APIs](/sdk/banned-apis) |
| **Guard Clauses** | `Throw.IfNull()` for clean argument validation |
| **Modern C#** | LangVersion `latest`, nullable enabled by default |
| **Deterministic** | Reproducible builds out of the box |

### Web SDK Features (ANcpLua.NET.Sdk.Web)

| Feature | Description |
|---------|-------------|
| **OpenTelemetry** | Logging, metrics, tracing with OTLP export |
| **Health Checks** | `/health` and `/alive` endpoints |
| **HTTP Resilience** | Retries, circuit breakers via Polly |
| **DevLogs** | Browser console to server logs |

### Test SDK Features (ANcpLua.NET.Sdk.Test)

| Feature | Description |
|---------|-------------|
| **xUnit v3** | Latest xUnit with Microsoft Testing Platform (MTP) |
| **Coverage** | Test coverage tooling pre-configured |
| **FakeLogger** | Opt-in extensions for logging tests |

## What Are the Requirements?

The SDK requires Central Package Management to be enabled:

```xml
<!-- Directory.Packages.props -->
<PropertyGroup>
  <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
</PropertyGroup>
```
